# Tomcat集群

  • 提高服务的性能，并发能力，以及高可用性
  • 提供项目架构的横向扩展能力
  • 推荐一台机器部署一个。多个Tomcat的话,性能下架比如，内存cpu等。
  • 多台服务成本高，项目很多少用户就不用了考虑了

# 实现原理

  • 通过Nginx负载均衡进行请求转发 查看了解更多

# 部署

一机一个Tomcat就不用修改配置，一机多个tomcat的就配置如下

添加新增的Tomcat相关 环境变量

CATALNA_BASE=C:\\Tomcat1
CATALNA_HOME=C:\\Tomcat1
TOMCAT_HOME=C:\\Tomcat1

CATALNA_2_BASE=C:\\Tomcat2
CATALNA_2_HOME=C:\\Tomcat2
TOMCAT_2_HOME=C:\\Tomcat2

第一个tomcat不变， 打开第二个tomcat目录bin下catalina.bat、catalina.bat进行替换修改

CATALNA_BASE -> CATALNA_2_BASE
CATALNA_HOME -> CATALNA_2_HOME

打开第二个tomcat的conf目录下server.xml，进行端口号prot修改

<Server prot="9005" shutdown="SHUTDOWN">

<Connector port="9080" protocol="HTTP/1.1">

分别进入两个Tomcat的bin目录，启动Tomcat执行 startup.bat

访问localhost：8080 和localhost：8080 就能看到两个Tomcat
