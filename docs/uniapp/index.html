# v-slot注意事项

☘️坑

现在版本对v-slot兼容性很差

非要用来的话

报错：nexpected attribute name, near generic... 这时候吧 name 名字换个就可以了

template 必须要组件里的第一层，不能再外层嵌套节点入div

slot 里微信不支持过滤器 和 调用方法处理 和 都不能 对数据进行过滤操作。解决方法：直接在数据源做处理

多个组件插槽走缓存问题：

小程序对相同组件里的slot会有缓存，所以在tab切换的时候会看到显示的都是最后一个组件。 解决方法，在组件内部多写几个slot name这样，父组件v-slot:到不同的name就不会被小程序走缓存

插槽无效问题：

name的命令只能单驼峰入shopA 而不能shopAdnT,还有名字不能太长，还有必须要传递接口item，不然for循环下只有第一个插槽能生效

# swiper

这是个动态指定高度的swiper示例：

<swiper class="swiper-box" @change="change" :current="current" :style="{height:swiperHeight?swiperHeight+'px':''}">
    <swiper-item v-for="(item ,index) in productList" :key="index" class="swiper-item">
        <view v-for="(item2 ,index2) in item" :key="item2" class="swiper-item-obj" >
            {{item2.money}}
        </view>
    </swiper-item>
</swiper>

swiperHeight 初始的时候不要有默认值，不然swiper-item-obj 取到的是填充后的高度

解决办法：先计算好高度，在给swiper赋值对应高度

methods:{
    chengSwiperHeight(){
        let parentWidth = 0
        let item = {}
        const query = uni.createSelectorQuery().in(this)
        
        query.select(".swiper-item").boundingClientRect(data => {
            parentWidth = data.width
            console.log("parentWidth", data)
        }).exec()
        
        query.select(".swiper-item-obj").boundingClientRect(data => {
            item.width = data.width
            item.height = data.height
            console.log("item", data)
        
            let rowNum = parseInt(parentWidth / item.width)
            let columNum = Math.ceil(7 / rowNum)
            
            this.swiperHeight = columNum*item.height
        }).exec()
    }
}

微信App支付功能申请

  1. 到 微信开放平台 申请移动应用并开通支付功能，申请应用后可以获取 AppID 和 AppSecret 值
  2. 应用接入 微信商户平台，选择 App 支付
  3. 开通支付功能后可获取支付业务服务器配置数据：PARTNER（财付通商户号）、PARTNER_KEY（财付通密钥）、PAYSIGNKEY（支付签名密钥）
  4. 需要将从微信开放平台申请的appid，填回到 manifest-App SDK配置-支付-微信支付 中。打包后生效。

微信小程序支付在 微信商户平台 申请支付时，选择公众号支付；普通浏览器里也可以调起微信进行支付，这个在微信叫做H5支付，此功能未开放给普通开发者，需向微信单独申请， 详见

# 绕审核

现在审核越来越严格了,简单的绕审核就是，前端全局枚举个版本号，在通过据库对这个枚举版本号进行布尔值控制来达到内容的显隐。

这样不会影响线上的版本8，提交审核的时候把当前最新版本号9置于false 来隐藏内容，当小程序审核通过后再置于true显示内容， 然后看是要小程序的灰度发布还还是直接发布即可。

# 键盘关闭后位置上移问题

function fitKeyboard() {
    var u = navigator.userAgent;
    var flag;
    var myFunction;
    var scrollTop;
    document.body.addEventListener('focusin', () => {  //软键盘弹起事件
        flag = true;
        scrollTop = document.documentElement.scrollTop;
        clearTimeout(myFunction);
    })
    document.body.addEventListener('focusout', () => { //软键盘关闭事件
        flag = false;
        if (!flag) {
            myFunction = setTimeout(function () {
                // console.log(scrollTop)
                if (scrollTop != 0) {
                    window.scrollTo({top: scrollTop, left: 0, behavior: "smooth"})//重点  =======当键盘收起的时候让页面回到原始位置(这里的top可以根据你们个人的需求改变，并不一定要回到页面顶部)
                }
            }, 200);
        } else {
            return
        }
    })
}
