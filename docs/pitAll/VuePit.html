# veu-cli 多个单页面入口

再在public下复制一下index.html,修改成admin.html用来对应游览的入口

然后在复制创建一个admin的main.js入口文件

最后在webpack.base.config.js 添加 admin页面路径 和对应的入口文件

module.exports = {
  pages: {
    //==页面1==
    index: {
      // page 的入口
      entry: 'src/index/main.js',
      // 模板来源
      template: 'public/index.html',
      // 在 dist/index.html 的输出
      filename: 'index.html',
      // 当使用 title 选项时，
      // template 中的 title 标签需要是 <title><%= htmlWebpackPlugin.options.title %></title>
      title: 'Index Page',
      // 在这个页面中包含的块，默认情况下会包含
      // 提取出来的通用 chunk 和 vendor chunk。
      chunks: ['chunk-vendors', 'chunk-common', 'index']
    },


     //==页面2==
    // 当使用只有入口的字符串格式时，
    // 模板会被推导为 `public/admin.html`
    // 并且如果找不到的话，就回退到 `public/index.html`。
    // 输出文件名会被推导为 `admin.html`。
    admin: 'src/admin/main.js' 
  }
  
}

# 利用cdn 减小包体积

下面是以query为例，同理我们也可以应用在其他的cdn资源， 如element ui 等

module.exports = {
    //configureWebpack 是cli 提供的对象，使用webpack-merge和并生效 
    configureWebpack:{ 
         
        //==下面的属性都是按webpack的api进行编写，详细的官网都有介绍==
        
        //防止将某些 import 的包(package)打包到 bundle 中,减小包体积
        externals: { 
            jquery: 'jQuery'
        },
        plugins:[
            //自动加载模块，而不必到处 import 或 require 
            new webpack.ProvidePlugin({
                $: 'jquery',
                jQuery: 'jquery'
            })
        ]

    }

}
