# 多线程

  • 内存中的运行程序即进程，一个程序一个进程，一个进程可以多线程

# 单线程

public static void main(String[] args) throws IOException {
    fn();//1 2 3....99
    System.out.println(Math.floor(2.25));
    // 打到99后才会 打印2.0  这就是单线程(一条流水线)
}

public static void fn() {
    for (int i = 0; i < 100; i++) {
        System.out.println(i);
    }
}

# Thread类

  • Java 虚拟机允许应用程序并发地运行多个执行线程。

# 重写run

多线程都需要子类重写run

public class TheadChiled extends Thread {
    public void run() {

        }

}

# start()

来启动多线程，注意只能启动一次

public static void main(String[] args) {
    TheadChiled th = new TheadChiled();
    th.start(); //启动
    for (int i = 0; i < 20; i++) {
        System.out.println("我是另一条线" + i);
    }
    
    //这样打印结果是混乱的因为 开了两条线程在工作
}

# 线程池

  • 因为没创建一个新线程，开销是相当大的，创建和销毁线程花费的时间和消耗的系统资源都相当大。 所以需要我们的线程池来解决
  • 线程池用线程工程来创建，下面就介绍下这些工程

# Executors类

////实现接口
public class RunnableChiled implements Runnable {
    public void run() {
        System.out.println(Thread.currentThread().getName());
    }
}


//实例
public static void main(String[] args) {
    ExecutorService exs = Executors.newFixedThreadPool(3); //3条鱼
    exs.submit(new RunnableChiled());
    exs.submit(new RunnableChiled());
    exs.submit(new RunnableChiled());
    //pool-1-thread-2 pool-1-thread-1  pool-1-thread-3
    // 证明线程并没有清理掉还在池里

    exs.shutdown();
    // 干掉线程池，一般不用，有池多好啊
}

# 线程异步计算

Callable 才有返回值

public class CallableChiled implements Callable<Integer> {
    private int a;

    //巧妙利用构造器传参
    public CallableChiled(int a) {
        this.a = a;
    }

    public Integer call() {
        int sum = 0;
        for (int i = 0; i < a; i++) {
            sum += i;
        }
        return sum;
    }
}

public static void main(String[] args) throws ExecutionException, InterruptedException {
    ExecutorService es = Executors.newFixedThreadPool(2);
    Future<Integer> f1 = es.submit(new CallableChiled(100));
    Future<Integer> f2 = es.submit(new CallableChiled(200));
    System.out.println(f1.get());//4950
    System.out.println(f2.get());//19900
    es.shutdown();
}

# synchronized()

利用同步代码块解决线程安全隐患:越安全速度越低

public class theadchiled implements Runnable {
    private int a = 100; 
    private Object obj = new Object();//必须造一个有名对象

    public void run() {
        while (true) {
            //线程共享数据，保证安全需要加入同步代码块
            synchronized (obj) {
                if (a > 0) { System.out.println(Thread.currentThread().getName() + "出售第" + a--);
                }

            }
        }
    }
}

利用同步方法解决隐患(推荐，代码量少)

public class theadchiled implements Runnable {
    private int a = 100;

    public void run() {
        while (true) {
            payTicket();
        }
    }

    private synchronized void payTicket() { //同步方法
        if (a > 0) {
            System.out.println(Thread.currentThread().getName() + "出售第" + a--);
        }
    }
}

# Lock接口 推荐

1.5后 synchronized 被替代了。

Lock 实现提供了比使用 synchronized 方法和语句可获得的更广泛的锁定操作。此实现允许更灵活的结构，可以具有差别很大的属性，可以支持多个相关的 Condition 对象。

# ReentrantLock 类

public class theadchiled implements Runnable {
    private int a = 100;
    private Lock lock = new ReentrantLock(); //创建实现类对象


    public void run() {
        while (true) {
            lock.lock();//上锁
            if (a > 0) {
                System.out.println(Thread.currentThread().getName() + "出售第" + a--);
            }
            lock.unlock();//解锁
        }
    }
}

# 死锁

多线程出现同步嵌套，线程进入同步需要获取锁,如果不出去就不会释放锁,都不释放就都死在那吧

死锁案例：可以配合死锁图片来理解

//A锁
public class lockA {
    private lockA() {}
    public static final lockA lockA = new lockA();
}


//B锁
public class lockB {
    private lockB() {}
    public static final lockB lockB = new lockB();
}



//死锁
public class deadLock implements Runnable {
    private int i = 0;
    
    public void run() {
        while (true) {
            if (i % 2 == 0) {
                //a-b
                synchronized (lockA.lockA) {
                    System.out.println("进入A");
                    synchronized (lockB.lockB) {
                        System.out.println("进入B");
                    }
                }
            } else {
                //b-a
                synchronized (lockB.lockB) {
                    System.out.println("进入B");
                    synchronized (lockA.lockA) {
                        System.out.println("进入A");
                    }
                }   
            }
        }
    }
}


//实例
public static void main(String[] args) {
    deadLock dead = new deadLock();
    Thread t1 = new Thread(dead);
    Thread t2 = new Thread(dead);

    t1.start();
    t2.start();

    //我们写的是死循环，会一直打印
    // 而运行结果打印了一会就没再打了，因为出现死锁
}

最终的好方法：等待与唤醒

public class resource {
    public String name;
    public String sex;
    
    public boolean flag = false;//创建一个控制器
}



public class inputt implements Runnable {
    private resource r;

    public inputt(resource r) {
        this.r = r;
    }

    public void run() {
        int i = 0;
        while (true) {
            synchronized (r) {
                if (r.flag) {//true
                    try {
                        r.wait();//等待
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                } else { //false
                    if (i % 2 == 0) {
                        r.name = "张三";
                        r.sex = "男";
                    } else {

                        r.name = "lisi";
                        r.sex = "nv";
                    }
                    i++;
                }

                r.flag = true;
                r.notify();//唤醒
            }
        }

    }
}



public class ouputt implements Runnable {
    private resource r;

    public ouputt(resource r) {
        this.r = r;
    }

    public void run() {

        while (true) {
            synchronized (r) {
                if (!r.flag) { //false
                    try {
                        r.wait();//等待
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                } else {//true
                    System.out.println(r.name + "。。。" + r.sex);
                }


                r.flag = false;
                r.notify();
            }
        }
    }
}




public static void main(String[] args) {

    resource r = new resource();//这样只new 一次就不会跟对象跑了
    inputt in = new inputt(r);
    ouputt out = new ouputt(r);

    Thread tin = new Thread(in);
    Thread tout = new Thread(out);

    tin.start();
    tout.start();
    //李四男  卧槽

}
