# 内部链接

以如下的文件结构为例：

.
├─ README.md
├─ foo
│  ├─ README.md
│  ├─ one.md
│  └─ two.md
└─ bar
   ├─ README.md
   ├─ three.md
   └─ four.md

[Home](/) <!-- 跳转到根部的 README.md -->
[foo](/foo/) <!-- 跳转到 foo 文件夹的 index.html -->
[foo heading anchor](/foo/#heading) <!-- 跳转到 foo/index.html 的特定 anchor 位置 -->
[foo - one](/foo/one.html) <!-- 具体文件可以使用 .html 结尾 -->
[foo - two](/foo/two.md) <!-- 也可以用 .md -->

# 导入代码段

是把整个文件的所有内容都展示

TIP

如果你更改引入文件的内容，页面没有热更新的话,可以把引入地址随便改一下，在改回来就能重新读取

 

 

 

const a = 1;
export default function () {
    console.log("花里胡哨111");
    console.log("花里胡哨2");
    console.log("花里胡哨3");
};

# 自动识别静态资源路径

<img :src="$withBase('/20151df123a.jpg?v=1')" width="100" alt="img">

TIP

$withBase() 可以帮助你生成正确的路径：Vue 组件中和Markdown 文件中都可以使用这就不怕base路径更改了
