# 搭建博客

# 为什么要自己搭建博客

很多技术人员会从博客园迁移到简书又迁移到掘金，因为书写与阅读、推广都有所提高。 自己选择的是有道云笔记，因为他的md支持性更好，缺点就是没有开源共享。

个人比较喜欢vue，看着作者写的官方文档也很舒服，归纳、整理、阅读、md拓展、md中使用vue等做得挺好的。 公开在GitHub上也能更好的推广自己,复习刷题也更有针对性。

# 搭建流程

跟着VuePress官方文档 走就可以了。可以跟官网一模一样，也可以根据自己习惯进行架构演进。

由于之前用hexo搭建了博客放 .github.io 根仓库里，所以不想动动他。 后面发现 GitHub 提供gh-pages静态文件页面游览分支，这样我们就可以放在别的仓库里了。

"itdoc"是我创建来存放VuePress博客项目,master主分支用来存放整个业务代码，gh-pages 用来存放打包后的静态文件。这样就能起到双层作用 。

这里我提供了自己的 初始化架构下载

<!--打开项目，然后安装vuepress 看你要全局还是项目依赖二选一 -->
<!--全局安装-->
npm install -g vuepress
<!--项目依赖安装-->
npm install -D vuepress

<!--下载剩余的项目依赖：配置了sass-->
npm install
<!--跑启动项目-->
npm run  dev

部署

TIP

提示：deploy.sh和deploy2.sh部署脚本里的仓库地址换成你自己

<!--部署到页面分支-->
npm run  deploy
<!--部署到主分支-->
npm run  deploy2

项目访问地址 ：https://<USERNAME>.github.io/<REPO>/

我的项目地址：https://abc123lzq.github.io/blog/`

# ssh公私钥匹配

由于发布需要ssh公私钥匹配，所以如你没还没匹配的话可以跟着流程走一遍。

创建一个 SSH key

$ ssh-keygen -t rsa -C "your_email@example.com"

🎉 生成 id_rsa 和 id_rsa.pub 两个秘钥文件,把C:\Users\Administrator\.ssh\id_rsa.pub里的公钥扔到你的github配置里

# authenticity验证失败

git clone或者push的时候，经常会报这样的错误：

WARNING

ERROR: The authenticity of host 'github.com (192.30.255.112)' can't be established. SA key fingerprint is SHA256:nThbg6kXUpJWGl7E Are you sure you want to continue connecting (yes/no)? yes

TIP

一定要选择 yes 才能生存know-host文件 这样就能解决验证失败问题
