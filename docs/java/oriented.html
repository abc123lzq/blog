# 编程思想

# 封装

  • 方法、类等，就是一个最基本的封装体
  • 提高复用，控制作用域

# 变量特点

  • 成员变量(类的全局变量)： 跟随对象在堆中存储，内存等待jvm清理
  • 私有变量： 外部没办法修改内部类的私有变量，可以调用内部类暴露的方法，让内部类自行修改
  • 局部变量： 跟随方法运行完毕后出栈
  • 变量： 都需要附上初始值，变量是有快作用域
public class Car {
    String color;//1. 成员变量
    private int price; //私有变量 (推荐)
    
    //set
    public void setPrice(int a) {
        if (a < 0 || a > 200) {
            price = 20;
        } else {
            price = a;
        }
    }
    //get
    public int getPrice() {
        return price;
    }
    
    //方法
    public void run() {
        int x ; //局部变量
        System.out.println(x); //2. 报错,变量没有赋值默认值
        
        for (int i = 0; i < 4; i++) {
             System.out.println(i); //1 2 3 4
        }
        System.out.println(i); //3. 报错，变量i是有快作用域

        String color = "五彩缤纷"; 
        System.out.println("属性就近原则 " + color);//"五彩缤纷"
        System.out.println("指向调用者本身 " + this.color);// 4. this指向
    }
}

# 继承

  • 格式： class子类 extends父类
//父类
public class Car {
    String name;
    public void sayName() {
        System.out.println(this.name);
    }
}

//子类继承
public class Bmw extends Car { };

//用例
Bmw bmw = new Bmw();
bmw.name = "宝马";
bmw.sayName();

不能继承多个父类

# 子类权限等级

  • 子类的权限要小于父类的权限
  • public 公共 > protected 受保护 > default 默认( 不写就是默认了) > private 私有

# 子类成员变量

  • 就近原则
  • this调用的是子类，super调用的是父类

# 子类重写

  • 子类中，出现了和父类一模一样的方法的时候，子类会重写父类的方法。
  • 函数的返回值类型 、函数名、 参数列表都要一样

使用场景

  • 比如不好对父类新增功能，就在子类上重写自己的业务功能
//父类
public class Car {
    public void run() {
        System.out.println("陆地行驶");
    }
}

//子类
public class Bmw extends Car { 
    public void run() {  //重写 
        System.out.println("海陆空行驶");
    }
}

# 多态

  • 创建类型与创建对象是有关联关系，如父子继承、接口与实现关系。
public class Parent {
    public void show() {
        System.out.println("父类里的方法");
    }
}

public class Child extends Parent {
    public void show() {
        System.out.println("子类里的方法");
    }
}

public class test {
    public static void main(String[] args) {
        Parent parent = new Child(); //多态 ：创建类型与创建对象是父子关系
        parent.show();//子类里的方法
    }
}

# 向上转型

Parent parent = new Child(); //把子类提升成父类
  • 方法，只能调用 父子共有的方法 ：子类有重写就用子类的方法，没有重写就找父类。如果父类也没有这个方法，就报错。
  • 变量，只会找父类：如果父类没有这个变量，就报错。

# 强制向下转型

Parent parent = new Child(); 
Child child = (child) parent; //父类强转子类
  • child多态对象，可以调用 父类 和 自己的 方法和变量。

# 构造方法

  • 方法的名字必须和类的名字一样
  • 构造方法不能返回值，也不能用void
  • new 对象的时候自动运行

创建对象并初始化赋值

public  class Web {
    private String name;
    private int age;

    //有参构造方法
    public Test(String name, int age) { //构造方法
        System.out.println("new对象的时候自动运行");
        this.name = name;//初始化赋值
        this.age = age;
    }
    
    // 无参构造方法
    public Web() {    
         this.name = "lzq";
         this.age = 18;
    }

    //get set 
}


public class Test {
    public static void main(String[] args) {
        Web web = new Web("lzq",20); //调用有参构方法
        web.getAge();//20
        
        Web web2 = new Web(); 
        web2.getAge();//18
    }
}

# super关键字

  • spuer. 能让当前子类使用父类型对象的变量、方法属性等。
  • spuer() 方法,可以触发父类的构造方法

spuer()方法

  • 当父类和子类都不写，都默认调用空参构造方法spuer()，但父类重写子类没重写就会报错
  • 子类上重写构造方法的时候记得加上spuer()，来触发父类的构造方法
  • 父类有多个重载构造方法，子类只需随便选一种就可以了
public class Parent {
    public Parent(String msg) { //重写的时候
        System.out.println("父类构造方法"+ msg);
    }
    public void sleep() {            
        System.out.println("睡觉");
    }
}

public class Child extends Parent {
     public Child(){  //重写的时候
         super('hello'); //必须第一行，触发父类的构造方法
         System.out.println("子类构造方法");
         super.sleep();
     }
}

public class test {
    public static void main(String[] args) {
        new Child();//父类构造方法hello   子类构造方法  睡觉
    }
}

# 修饰符

# 权限修饰符

  • 权限由大到小排序： public > protected > default > private

# final修饰符

最终形态、不可变的修饰符：可以修饰类、类成员、以及局部变量

  • final修饰的类不能被继承，也不能有子类
  • final修饰的方法：子类可以继承，但不能重写
  • final修饰的变量：就变成常量

基本类型那数值就终身不变

引用类型那引用地址就终身不变，但可以改地址里面的属性

  • final修饰的成员变量：必须赋值
final int age = 20;

也可以在构造方法来赋值，但只能赋值一次

public class a3 {
    final int age;

    public a3(int age) {
        this.age = age;//在这边赋值
    }
}

# static修饰符

  • static修饰的变量、方法、代码块都是隶属于类(class)级别的,跟对象无关。
  • 会被提升到该类的全局共享数据，并且优先解析
  • 某一类物体如果可以被多个其他物体所共享，那么可以将这类物体使用static修饰。
  • private static修饰的对象只能被本类中的方法（静态或非静态的）调用。
class Employee{
    static String company = "阿里巴巴"; //优先解析,并提升到该类全局共享数据
};


//==使用==
Employee e = new Employee();

//对象名.静态方法名，工作中不这样使用
System.out.println(e.company);
//类名.静态方法名的工作中使用这种写法
System.out.println(Employee.company);

e = null;
System.out.println(e.company);//不会报空指针，说明静态变量跟new对象无关。
  • 静态只能调用静态

静态解析优先于非静态，所以也不能配合this和super man()方法里，也是通过new对象才能，去访问非静态。

public class Person {
    String name;
    int age;

    public static void function() {
        System.out.println(name + age); //报错：找不到
    }
}
  • 静态语句块在类加载阶段执行，并且只执行一次，并且是自上而下的顺序执行，在构造方法之前执行。
public class Car{
    static{
        System.out.println("静态语句块1");
    }

    static{
        System.out.println("静态语句块2");
    }
}

# 多态中的静态

  • 运行静态方法，是运行父类的，
  • 运行非静态方法，是运行子类重写的方法
  • 成员变量，全部看父类
//父类里show是静态方法，所以不会运行子类的重写
public static void Show(){
      System.out.println("父类静态方法");
}

# 抽象类

  • 只要存在一个，没有方法体的都是抽象类,须要用abstract来修饰才不会报错
  • abstract修饰的类，也是抽象类
public abstract class Develop { //abstract修饰
    public abstract void work(); //没有方法主体 abstract修饰
}

# 没有方法体的，子类必须重写

public class Web extends Develop {
    public void work() {
        System.out.println("前端在开发界面");
    }
}

# 不能使用哪些修饰符

private abstract void work();  //私有 报错
static  abstract void work();  //静态 报错
final   abstract void work();  //最终  报错

# 匿名对象

  • 只有创建对象语句，却没有把对象赋值给某个变量就叫做匿名对象
  • 匿名对象创建一次只能使用一次,而有名对象可以通过.进行无线调用
  • 匿名对象也不出一无是处，他可以当做参数来使用
//匿名对象 ,只能使用一次
int number=new Scanner(System.in).nextInt();  

//如果想多次监听键盘输入就得使用有名对象
Scanner sc=new Scanner(System.in);
sc.next();
sc.nextInt();

# 内部类

  • 内部类可以使用外类成员包括私有
  • 外部类要使用内部类的成员，必须先创建内部类对象
  • 内部类的同名变量调用: 就近原则
public class a1 {
    private int a = 1;

    class a11 {
        public void bbb() {
            System.out.println(a);//1 内部访问外部
        }
    }
}

public class test {
    public static void main(String[] args) {
        //创建内部对象
        a1.a11 tx = new a1().new a11(); 
        tx.bbb();//1
    }
}

public class a1 {

# 匿名内部类

可以简化：把实现类、重写方法、new实现类对象，合成一步

public interface Myint {
    public abstract void smoking();
}


//使用方式：  new 对象.方法 
new Myint() {  
    public void smoking() {
        System.out.println("大佬抽根烟");
    }
}.smoking();

因为匿名只能使用一次，所以保存到变量就可以无限使用

public abstract class Person {
    public abstract void eat();
    public abstract void sleep();
}


public class test {
    public static void main(String[] args) {
        Person person = new Person() {  //右边这块匿名内部类：其实是子类 
            public void eat() {
                System.out.println("吃饭");
            }
            public void sleep() {
                System.out.println("睡觉");
            }
        };

        person.eat();
        person.sleep();
    }
}

# 代码块{ }

  • 有块作用域 ，比构造器优先解析
  • 代码块和构造方法 new一次就执行一次。 静态代码块，只能执行一次
public class Person {
    private int age;

    public Person(int age) {
        this.age = age;
        System.out.println("3构造方法");
    }

    {
        System.out.println("2代码块，比构造器先运行");
    }

    static {
        System.out.println("1静态代码块，最牛我最快");
    }

}

# interface 接口

  • 不能带有方法体，固定格式：public abstract void
  • 变量：必须常量，固定格式：public static final

创建interface类型的MyInt接口文件

public interface MyInt {
    public abstract void say(); //不带方法体的抽象类
    public static final int a = 1;//变量必须常量
}

# 实现类

  • 实现类使用implements 接口，也称为接口的实现类
  • 实现类没全部重写抽象方法，那他还是一个抽象类。必须加abstract来修饰才不会报错
  • 实现类可以接入多个接口
  • 实现类还能继续使用继承
public class UseInit extends Car implements Myint1, Myint2 {
      public void say1() {
           System.out.println("实现接口1");
       }
       public void say2() {
           System.out.println("实现接口2");
       }
}

# 接口与接口关系

  • 接口与接口之间只能继承，一个接口可以同时继承多个接口
public interface A extends B,UseInit{
    //...
}
