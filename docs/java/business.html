# Throwable异常类

  • Java 语言中所有错误或异常的超类。
  • 有三个子类 Exception (异常) 、 RuntimeException(运行异常) 、 Error （报错）

# Exception 类

at thorewa.deomsdf.main(deomsdf.java:6)

Exception 类 在main方法中 lang包里的数组索引 3 越界异常 在我们的代码文件中的第 6 行处

# 抛异常

throws 和 throw new

public static void main(String[] args) throws Exception {
    int[] arr = null;
    int i = getArray(arr); //运行着虚拟机处理不来，所以方法上也要加throws
    System.out.println(i);
}
    
//Exception in thread "main" java.lang.Exception: 数组为null

public static int getArray(int[] arr) throws Exception {
    if (arr == null) {
        throw new Exception("数组为null");
    }
    int i = arr[arr.length - 1];
    return i * 2;
}

# 异常处理

  • try{...运行}catch{...有异常走这里}finally{..必须要执行的代码}
 public static void main(String[] args) {
        int[] arr = {};
        try {
            int i = getArray(arr);
            System.out.println(i);
        } catch (ArrayIndexOutOfBoundsException err) {
            System.out.println(err);//不会报错只会打印错误
            //java.lang.ArrayIndexOutOfBoundsException: 越界咯 
            //原理：就是把错误抛给catch去捕获处理
        }
        
        System.out.println("我还是能运行的")//我还是能运行的
 }

public static int getArray(int[] arr) {
    if (arr == null) {
        throw new NullPointerException("空指针喽");
    }

    if (arr.length < 3) {
        throw new ArrayIndexOutOfBoundsException("越界咯");
    }
    return arr[3] + 1;
}

多态异常处理：说白的就是想写几个异常及写几个

public static void main(String[] args) {
        int[] arr = new int[0];
        try {
            int i = getArray(arr);
            System.out.println(i);
        } catch (NullPointerException err) {
            System.out.println(err);
        } catch (ArrayIndexOutOfBoundsException err) {
            System.out.println(err);
            //java.lang.ArrayIndexOutOfBoundsException: 越界咯
        }
    }

public static int getArray(int[] arr) throws NullPointerException, ArrayIndexOutOfBoundsException {
    if (arr == null) {
        throw new NullPointerException("空指针喽");
    }

    if (arr.length < 3) {
        throw new ArrayIndexOutOfBoundsException("越界咯");
    }
    return arr[3] + 1;
}

# HandlerExceptionResolver

当前端访问接口，服务500的时候浏览器就会显示报错信息，我们可以用全局异常包装处理返回一个json数据，规避一些不必要的信息泄露。

@Slf4j
@Component //注入到springmvc
public class ExceptionResolver implements HandlerExceptionResolver {
    @Override
    public ModelAndView resolveException(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse, Object o, Exception e) {
        log.error("{}Exception", httpServletRequest.getRequestURI(), e);
        ModelAndView modelAndView = new ModelAndView(new MappingJacksonJsonView());
        modelAndView.addObject("status", ResponseCode.ERROR.getCode());
        modelAndView.addObject("msg", "接口异常");
        modelAndView.addObject("data", e.toString());
        return modelAndView;
    }
}
