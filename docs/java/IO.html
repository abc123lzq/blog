# FileFilter 接口

文件顾虑器,没有实现类，需要自己手动写

实现类，implements接入，然后alert+回车选择自动填写方法

public class fileflierd implements FileFilter {
    @Override
    public Boolean accept(File pathname) {
        return false;
    }
}

//改写过滤方法
public Boolean accept(File pathname) {
    String name = pathname.getName();
    return  name.endsWith(".txt"); //返回 结尾是.text 的
}
    
//开始过滤
File f1 = new File("c:\\abc");
File[] fileArr = f1.listFiles(new fileflierd());
for (File ff:fileArr) {
    System.out.println(ff);
    //c:\abc\1.txt
    //c:\abc\2.txt
}

递归方法遍历全目录

public static void main(String[] args) {
    File f1 = new File("c:\\abc");
    getAllList(f1);

}

public static void getAllList(File dir) {
    File[] fa = dir.listFiles();
    for (File f : fa) {
        if (f.isDirectory()) {
            //是文件的话就递归
            getAllList(f);
        }
        System.out.println(f);
    }
}

# 字节流

硬盘流到java系统就是输入， java系统流到硬盘就是输出(主体立场是java系统)

# OutputStream类

  • 此抽象类是表示输出字节流的所有类的超类。
  • 每次只操作文件中的1个字节
  • 子类如FileOutputStream 才能new 对象
public static void main(String[] args) throws IOException {
    //需要用子类来创对象
    FileOutputStream outp = new FileOutputStream("c:\\abc\\1.txt");
    outp.write(100); //没有就创建，有就覆盖内容
    //100转二进制01011010  的编码表是 d
    
    
    //**写字节数组**
    byte[] bb = {-11, -22, -33};
    outp.write(bb);//负数是汉字 蹶?
    //只写一部分(开始索引，要几个)
    outp.write(bb,1,2);//负数是汉字 蹶
    
    //***String的getBytes()***
    outp.write("直接写字符串".getBytes());
        
    outp.close();
}

# FilterInputStream类

read()

一次一个字节，执行一次就会自动读写一下个字节,到达数据结尾返回-1

FileInputStream f1 = new FileInputStream("c:\\abc\\1.txt");
int i = f1.read();
System.out.println(i);//100
f1.close();

read的特性会自动跳到到下个字节出所以才会多此一举用len变量来记住当前的

FileInputStream f1 = new FileInputStream("c:\\abc\\1.txt");
int len = 0; //接收read方法的返回值如果是-1就表示读完了
while ((len = f1.read()) != -1) {
    System.out.println(len);//100 99 102
    System.out.println((char) len);//d c f
}
f1.close();

读取字节数组：指定数组容器的长度，把字节 按长度一批一批的替换进去

FileInputStream f1 = new FileInputStream("c:\\abc\\1.txt");
//创建字节数组
byte[] b = new byte[2]; //长度为2的容器

int len = f1.read(b);
System.out.println(b);//[B@6a38e57f
System.out.println(new String(b));//dc
System.out.println(len);//2 ：表示读取到了多少个有效字节


len = f1.read(b);
System.out.println(len);//1  ：是剩下的字节个数
// 上个状态是dc，因为只剩一个f了 所以就把d替换了f 最终就是fc
System.out.println(new String(b));//fc ：read一次就自动到下一个字节

len = f1.read(b);
System.out.println(len);//-1 是虚拟机返回
System.out.println(new String(b));//fc  值就是最后的最不变

len = f1.read(b);
System.out.println(len);//-1
System.out.println(new String(b));//fc

遍历读取数组

FileInputStream f1 = new FileInputStream("c:\\abc\\1.txt");//abcdefg
//创建字节数组
byte[] b = new byte[2]; //标准的是1024字节也就是1kb

int len = 0;
while ((len = f1.read(b)) != -1) {
    System.out.println(new String(b));//ab   cd   ef gf
    System.out.println(new String(b, 0, len)); //ab cd ef g  :这才是标准的读取
}
f1.close();;

# 文件复制

  • 说白了就是读写，结合上面介绍的两个就可以实现
  • 读一个写一个速度是非常的慢龟速 （不推荐）
public static void main(String[] args) {
    FileOutputStream fo = null;
    FileInputStream fi = null;

    try {
        fi = new FileInputStream("c:\\abc\\1.txt");
        fo = new FileOutputStream("c:\\abc\\2.txt");

        int len = 0;
        while ((len = fi.read()) != -1) {
            fo.write(len);
        }
    } catch (IOException ex) {
        System.out.println(ex);
    } finally {
        try {
            fi.close();
            fo.close();
        } catch (Exception EX) {
            System.out.println(EX);

        }
    }
}

采用字节数组来缓冲提高效率 （推荐）

try {
    fi = new FileInputStream("c:\\abc\\1.txt");
    fo = new FileOutputStream("c:\\abc\\2.txt");
    byte[] bytes = new byte[1024];
    //如果都是大文件可以[1024*10],这是牺牲内存换速度，所以要看情况选值
    
    int len = 0;
    while ((len = fi.read(bytes)) != -1) {
        fo.write(bytes, 0, len);
    }
}

# 对象的序列化与反序列化

  • 一直new对象会牺牲内存，序列化可以牺牲硬盘来换内存非常的值得
  • 对象中的数据，以流的形式，写入到文件中保存的过程称为写对象或对象的序列化

# ObjectOutputStream类

将对象写到文件中，实现序列化

//java.io.NotSerializableException: 提示没有Serializable

//person类 需接入接口 implements Serializable
public class person implements Serializable { }


//序列化类
public static void f1() throws IOException {
    FileOutputStream fos = new FileOutputStream("c:\\abc\\person.txt");
    ObjectOutputStream oos = new ObjectOutputStream(fos);
    person p = new person("lzq", "4");
    oos.writeObject(p);
}

# ObjectInputStream类

将序列化文件对象读取出来

//需要抛错ClassNotFoundException
public static void f2() throws IOException, ClassNotFoundException {
    FileInputStream fis = new FileInputStream("c:\\abc\\person.txt");
    ObjectInputStream ois = new ObjectInputStream(fis);

    //读取对象
    Object obj = ois.readObject();
    System.out.println(obj);
    //person{name='lzq', age='4'}
    ois.close();
}

# 序列号冲突问题

我们改写类的源码的时候，编译后的序列号就会随之更改了，这样新旧序列号就不一样，从而报序列号冲突

# 自定义序列号来解决冲突

做一个终身不变的序列化就可以,注意编译后也不要去改了

private String name;
private String age; //阻止成员变量序列化：只能修饰成员变量
private static final long serialVersionUID = -6849794470754667710L;//自定义序列号

# 打印流

PrintStream 和 PrintWriter

# 特点

  • 此流不负责数据源，只负责数据目的。
  • 为其他输出流添加功能。
  • 永远不会抛出IOExceotiaon但是可以抛出别的异常
  • PrintWriter比PrintStream多了一个接收字符输出流

# 打印流输出目的是File对象

public static void main(String[] args) throws FileNotFoundException {
    File f = new File("c:\\abc\\1.txt");
    PrintWriter pw = new PrintWriter(f);
    pw.println(100); //100 长什么样就是怎么样
    pw.write(100);//d
    pw.close();
    //文件打印了100  d
}

# 输出语句是char数组

public static void main(String[] args) throws FileNotFoundException {
    int[] ints = {1, 2};
    System.out.println(ints); //[I@6a38e57f

    byte[] bytes = new byte[1024];
    System.out.println(bytes);//[B@5577140
    
    char[] chars = {'a', 'b'};
    System.out.println(chars);//ab
}

# 打印流输出目的是String和流对象

    FileWriter fw = new FileWriter("c:\\abc\\2.txt");

# 打印流开启自动刷新

  • 输出的数据目的必须是流对象 OutputStream Writer
  • 必须调用println，printf，format三个方法中的一个，来启用自动刷新
 // 因为这里自动转成了流对象 File
FileOutputStream fos = new FileOutputStream("c:\\abc\\1.txt");

PrintWriter pw = new PrintWriter(fos);
pw.println("l");
pw.println("z");
pw.println("q");
pw.close(); //l z  q  自动刷新了
