# 性能优化

# shouldComponentUpdate

我们之前介绍过 父组件的render运行，子组件也会跟着运行，还有在运行render之前会先运行shouldComponentUpdate 返回false的话render就不会再执行 利用这些原理

当父组件的sate更改的时候，如果子组件业务逻辑上不需要重新render，就在子组件上的shouldComponentUpdate 返回false，

如果子组件需要重新render的时候就用利用props更新来触发子组件的render。 这样就可节省不必要的开销

shouldComponentUpdate(nextProps, nextState, nextContext) {
    if(nextProps.content!=this.props.content){
        return true
    }else {
        return false;
    }
}

# useEffect

useEffect给函数组件增加了操作副作用的能力。 它跟 class 组件中的 componentDidMount、componentDidUpdate 和 componentWillUnmount 具有相同的用途，只不过被合并成了一个 API。

 




 






 







 









import React, { useState, useEffect } from 'react';
import myAPI from 'myAPI';

function FriendStatusWithCounter(props) {
  const [count, setCount] = useState(0);

  // 具有 componentDidMount 和 componentDidUpdate的调用机制:
  useEffect(() => {
    document.title = `You clicked ${count} times`;
  });

  const [isOnline, setIsOnline] = useState(null);
    
  //可以定义多个
  useEffect(() => {
    //componentDidMount
    function handleStatusChange(res) {
      setIsOnline(res.isOnline);
    }

   myAPI.subscribeToFriendStatus(props.friend.id, handleStatusChange);

     // 具有 componentWillUnmount的调用机制:
    return () => {
     myAPI.unsubscribeFromFriendStatus(props.friend.id, handleStatusChange);
    };
  },[]);  // 监听空数组的时候

}

总结： useEffect(fn) 中 fn相当于在 componentDidMount、componentDidUpdate的时候运行

如果里面有 return函数的话，那函数相当于在componentWillUnmount的时候运行

useEffect(fn,data)

第二个参数data，用来监听对应数据变化后，才触发执行fn

//默认是初始化 和 props、state 数据变化的时候就会触发

    document.title = `You clicked ${count} times`;
});

//只有count变化，才会触发
useEffect(() => {
    document.title = `You clicked ${count} times`;
},count);

//只会触发一次，因为监控的数据没有变化
useEffect(() => {
    document.title = `You clicked ${count} times`;

不要过度依赖 useMemo

useMemo 本身也有开销。useMemo 会「记住」一些值，同时在后续 render 时，将依赖数组中的值取出来和上一次记录的值进行比较， 如果不相等才会重新执行回调函数，否则直接返回「记住」的值。 这个过程本身就会消耗一定的内存和计算资源。因此，过度使用 useMemo 可能会影响程序的性能。

在使用 useMemo 前，应该先思考三个问题：

  1. 传递给 useMemo 的函数开销大不大？ 有些计算开销很大就需要「记住」它的返回值，避免每次 render 都去重新计算。

  2. 返回的值是原始值吗？ 如果计算出来的是基本类型的值（string、 boolean 、null、undefined 、number、symbol），那么每次比较都是相等的，下游组件就不会重新渲染； 如果计算出来的是复杂类型的值（object、array），哪怕值不变，但是地址会发生变化，导致下游组件重新渲染。所以引用类型需要「记住」这个值。

  3. 如果自定义 Hook 中暴露出来的值是 object、array、函数等，暴露的引用类型都应该使用 useMemo 。以确保当值相同时，引用不发生变化。

# useRef

useRef可以“跨渲染周期”保存数据，不挂载ref默认取当前组件




 
 
 
 
 
 
 
 
 
 
 
 
 





 
 





















function App() {
    const [count, setCount] = useState(0);

    //用法一可以有效的保留数据原生地址,从而达到了有效清除定时器
    // let it; 无法清除，因为每次触发都会创建新的句柄，所以并没有清除到上一个it
    let it = useRef()
    useEffect(() => {
        it.current = setInterval(() => {
            setCount(count => count + 1)
        }, 1000)
    }, []) //只执行一次
    useEffect(() => {
        if (count > 10) {
            clearInterval(it.current)
        }
    })

    const counterRef = useRef();
    const onclikFn = useCallback(() => {
        setCount((count) => count + 1);
    
        //用法二能访问到子组件的属性
        counterRef.current.speak(); 
    }, [counterRef]);
    return (
        <div>
            <Foo ref={counterRef} onClick={onclikFn}></Foo>
        </div>
    )
}

class Foo extends Component {
    speak() {
        console.log(`now counter is ${this.props.count}`)
    }

    render() {
        return (
            <div>${this.props.count}</div>
        )
    }
}

# Hook 规则

  1. 不要在循环，条件或嵌套函数中去调用Hook， 确保总是在你的 React 函数的最顶层调用他们。

  2. 只在React函数中调用 Hook不要在普通的 JavaScript 函数中调用 (因为Hook是react自己封装的api)

# Hook常用技巧

useRef()

不仅可以用于 DOM refs，[ref」 对象是一个 current 属性可变且可以容纳任意值的通用容器, 所以我们经常用来存储实时对象 如定时器 或 上一轮的 props 或 state 或者 全局实时变量等。

function Counter() {
       const [count, setCount] = useState(0);
   
       const intervalRef = useRef();
       const intervalRef2 = useRef();
   
       //记住计算结果的值
       const double = useMemo(() => {
           return count * 2
       },[count])
   
       useEffect(() => {
           
           const id = setInterval(() => {
               setCount((prev) => prev + 1)
           }, 3000);
   
           //记住任何类型的值
           intervalRef2.current = id;
           intervalRef.current = count;
           console.log(intervalRef, intervalRef2)
   
           return () => {
               clearInterval(intervalRef2.current);
           };
       });
   
       return (
           <div>
               上次count的值：{intervalRef.current}
               <br/>
               当前count的值： {count}
               当前double的值： {double}
           </div>
       )
}

useReducer()

可以用来用做个一个增长的计数器来在 state 没变的时候依然强制一次重新渲染：

const [ignored, forceUpdate] = useReducer(x => x + 1, 0);

function handleClick() {
    forceUpdate();
}

# 组件运行机制

首次加载或者props和state变更的时候，执行栈就会从上往下执行执行宏任务的同步代码， 完后，事件循环执行宏任务中的异步代码（如生命周期函数或者计时器等异步方法），掌握好运行机制才能更好的控制住业务代码

function ListItem(props) {
   const [keyword, setKeyword] = useState(null)
   setKeyword([1,2,3])//这样就会死循环不停触发重新加载
   
   //如果父组件的回调有更改传入组装的props那子组件就会重新加载
   // 如果不是必要的props可以去掉减少开销
   props.cb()
}
