# 结合Redux

有的时候在屏幕跳转路由的时候屏幕里的组件并不会被销毁,所有redux更新的时候它也会触发逻辑，这时候我们可以利用navigation的生命周期来控制 比如通多是不是离屏状态。

let isfocus = React.useRef(false);
let SaveweeklyList = React.useRef([]);

React.useEffect(() => {
    const unsubscribe = navigation.addListener('focus', () => {
        isfocus.current = false;
        console.log('进入页面');
    });

    const subscribe = navigation.addListener('blur', () => {
        isfocus.current = true;
        console.log('离开页面');
    });
    
    //==mount==
    set_WEEKLY({data: false, key: 'isLoading'});

    // componentWillUnmount
    return () => {
        unsubscribe;
        subscribe;
    };
}, []);


let MyweeklyList = React.useMemo(() => {
    if (!isfocus.current) {
        let currentVal = SaveweeklyList.current;
         SaveweeklyList.current = currentVal.concat(weekly.weeklyList);
    }
}, [weekly.weeklyList]);//redux里的数据

# 调试

# Chrome 开发者工具调试

  1. ctrl+m 打开设置，开启debugger
  2. http://localhost:8081/debugger-ui

由于Chrome 调试目前无法观测到 React Native 中的网络请求，[下载react-native-debugger]来调试更合适

  1. 关掉http://localhost:8081/debugger-ui
  2. 打开react-native-debugger
  3. ctrl+m 打开设置，开启debugger

# 真机调试

手机开启开发者模式，链接数据线

#可以查看运行的设置
adb devices 

#设定从设备到电脑的端口转发
adb reverse tcp:8081 tcp:8081 

#跑起项目 
react-native run-android 

#也可以指定运行的设备
react-native run-android --deviceId c865e625//

#如果安装的时候提示已安装应用签名不同 ，卸载应用看看，还是提醒的话就用adb去卸载
adb uninstall com.awesomeproject
  1. 配置的应用名(可以根据com. 这来搜索)
public static final String APPLICATION_ID = "com.awesomeproject";

进入应用后 手机摇一摇就可以出现菜单，就可以选择debug模式

TIP

如果你已经安装apk，那真机调试的时候需要配置不同的包名，不然会报错，因为包名已经在手机上安装了， 如果相同包名，不同签名那就安装不了的会报错。

# 常见问题

# 线上环境图片不显示问题

网上查资料说是 IOS 9 以上图片资源链接要是https 才可以正常显示，

如果是 http 的链接则要设置 android\app\src\main\AndroidManifest.xml

<application
      android:usesCleartextTraffic="true" //添加这句
      android:theme="@style/AppTheme">
      {/*...*/}
</application>
