# 浮动
🔥hot

浮动元素会脱离文档流并向左/向右浮动，直到碰到父元素或者另一个浮动元素。

# 脱离文档

浮动会脱离文档,造成高度塌陷

# 内联排列

浮动会向左/向右浮动，直到碰到另一个浮动元素为止

TIP

如果父盒子没有定义高度，那它的高度取决于里面子盒子高度累积的高度，

由于浮动的子盒子的高塌陷，相对也会照成父盒子的高度塌陷。

# 清除浮动

1.在父盒子上使用clearfix 推荐

//全浏览器通用的clearfix方案【推荐】
// 引入了zoom以支持IE6/7
// 同时加入:before以解决现代浏览器上边距折叠的问题
.clearfix:before,
.clearfix:after {
    display: table;
    content: " ";
}
.clearfix:after {
    clear: both;
}
.clearfix{
    *zoom: 1;
}

2.BFC清除浮动

BFC 全称为 block formatting context，中文为“块级格式化上下文”。相对应的还有 IFC， 也就是 inline formatting context，中文为“内联格式化上下”。不过 IFC 作用和影响比较隐晦,所以掌握BFC即可

BFC可以包含浮动；通常用来解决浮动父元素高度坍塌的问题

给父元素设置overflow:auto来简单的实现BFC清除浮动.

但是为了兼容IE最好用overflow:hidden。但是这样元素阴影或下拉菜单会被截断，比较局限。

.partent-box{
  //overflow:auto
  overflow: hidden;
}

# 渐进增强和优雅降级

这只是个概念

# 渐进增强

一开始就针对低版本浏览器进行构建页面，完成基本的功能,然后再针对高级浏览器进行效果、交互、追加功能达到更好的体验。

# 优雅降级

一开始构建站点的完美特效功能，然后针对低版本浏览器进行兼容修复和降低特效，直到低版本浏览器能正常预览。

举例

写css3属性时，兼容性的写法顺序不太一样，比如transition属性，有些把transition放在前面有些是放在后面

.transition { /*渐进增强写法*/
  -webkit-transition: all .5s;
     -moz-transition: all .5s;
       -o-transition: all .5s;
          transition: all .5s;
}
.transition { /*优雅降级写法*/
          transition: all .5s;
       -o-transition: all .5s;
     -moz-transition: all .5s;
  -webkit-transition: all .5s;
}

</style>

# transform

transform 属性允许开发者无需借助 SVG、Canvas、Javascript 等方式，仅仅通过 CSS 便能以尽可能简单的方式实现对指定标签元素的旋转（rotate）、缩放（scale）、倾斜（skew）、平移（translate）等操作

transform: rotate(45deg);
transform: scale(0.1);
transform: skew(40deg,45deg);
transform: translate(-50%,-50%);

# calc() 计算单位

abc{
    //注意空格隔开
    width: calc(100% - 140px);
    height: calc(100vh - 10%);
}

苹果的刘海屏和底部的横条挡住界面的时候可以用计算方式去整体界面布局

.uni-page-body {
    padding-bottom: calc(140upx + env(safe-area-inset-bottom)) !important;
}

iOS11 新增特性，Webkit 的一个 CSS 函数，用于设定安全区域与边界的距离，有四个预定义的变量：

#安全区域距离左边边界距离
safe-area-inset-left 

#安全区域距离右边边界距离
safe-area-inset-right

#安全区域距离顶部边界距离
safe-area-inset-top

#安全区域距离底部边界距离
safe-area-inset-bottom

# 常见

pre 元素可定义预格式化的文本。被包围在 pre 元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。

如果内容比较长，div就会被拉长 我们可以定个宽在加上自动折行

pre {
    width: 500px;
    white-space: pre-wrap;
    word-wrap: break-word;
}

<div class="tDiv">
    <pre>{{ content}}</pre>
</div>

    left: 0;
